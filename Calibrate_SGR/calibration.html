   Project Title

SGR1806-20 Flare Data Calibration



Research Group:           Columbia Experimental Gravity (GECo)

Primary Researcher:    Rubab Khan

Research Advisor:         Luca Matone                                 

                
Project Goal:


As a part of the broader project of analyzing LIGO data for SGR1806-20 Flare event that occurred on December 27, 2004 during LIGO Hanford 4km detector's astrowatch run, this project verifies the data calibration at the relevant time and site.

Project Abstract:

The event gps-time is 788218239 (Astrowatch, A3) and the calibration gps-time is 784716588 (Engineering Run, E11). The list of channels is obtained from Conlog. Calibration data at the event time and calibration time are compared for inconsistency check.


General Outline:

The matlab script reads the relevant L0 data frames H-R-784716576-32.gwf and H-R-788218208-32.gwf for the intended gps-times and channels, and compares the first entry for those channnels (regardless of sampling rate) for consistency. It reports inconsistent readings and non-existant channels separtely. Also, when a channel is not found in the given L0 frames, the script also looks into the relevant minute-trend and second-trend frames: H-M-784713600-3600.gwf, H-M-788216400-3600.gwf, H-T-784716540-60.gwf, and H-T-788218200-60.gwf.

This code is very generic, and can be re-used to verify calibration at any two gps-times. For details, please refer to the code documentation.

Note: To get these old data frames, please scp them from CIT arvhive, and not LHO archive. These data are no longer on-line at LHO, so any attempt to read() from them should return ENODEV (for No Such Device), i.e., the tape volume is in a cabinet on site rather than in the robot. (However, remotely using scp to copy these files from fortress or other LHO machines does create some confusion).

Latest:

The matlab script, inconsistency report, non-existant channel report, L0 ffl file, minute-trend ffl file, second-trend ffl file, inquired channels list, relevant shell script, and condor script are available. Easy to reuse version of the inconsistency report is also available which have been produced by this script. Please check the code for details on these files.

Please note that the (frame file list) ffl files have to be recreated by individual users as it is specific to their file system. Locally copying the two relevant frames is recommended as it ensures faster access. The code is not compiled, and so the shell script and condor script presumes matlab installation and license availablity.
